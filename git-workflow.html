Clone

Este comando serve para clonar o repositório remoto para usá-lo localmente.

Como usar?

Para termos acesso ao repositório primeiro precisamos clona-lo, isto é, copiar do Github para nosso computador. Desta forma teremos 2 repositórios, um local e outro no Github. Para clonar é preciso:

1. Acessar seu repositório e copiar o HTTPS disponível.

2. Criar uma pasta no seu sistema onde gostaria de armazenar seu novo projeto de programação.

3. Dentro desta pasta clique com botão direito e selecione “Abrir no Terminal”.

4. Com o terminal aberto escreva o comando: git clone [HTTPS-do-repositório]

Branch

Este comando permite criar, listar, renomear e excluir ramificações. Mas nesse site vamos ensinar apenas como criar uma branch no repositório.

Como usar?

Com Git branches, é possível criar diferentes branches de desenvolvimento que podem convergir no mesmo repositório. Em qualquer projeto Git, podemos visualizar todos os branches digitando o seguinte comando: [git branch]

1. Se nenhum branch tiver sido criado, o terminal não vai mostrar nenhuma informação. Não se preocupe, criar um branch é realmente simples: É só digitar [git branch *nome_da_branch*]

2. Pronto, agora você tem outra branch além da main. Se quiser visualizar quais branches você tem ou em qual está, pode usar o comando [git branch].

Commit

Este comando serve para gravar as mudanças permanentemente no histórico de versão com uma mensagem.

Como usar?

Cada commit tem uma mensagem de EXTREMA importância, a mensagem deve ser objetiva e coerente com as mudanças realizadas. Quando outras pessoas ou mesmo quem fez o commit olhar depois de meses, a mensagem deve explicar o que foi feito. Dessa forma, é preciso:

1. Após fazer modificações no código, abrir o terminal no VS Code e digitar o comando [git commit -m "Mensagem importante"].

2. O parâmetro [-m] serve para passarmos uma mensagem de commit, que será incluído entre aspas.

Merge

Este comando é usado pelo git pull para incorporar alterações de outro repositório e pode ser usado manualmente para mesclar as alterações de uma ramificação para outra.

Como usar?

Para realizar um merge navegue para a branch main e execute o comando: [git merge *nome-da-branch-desejada*]

1. Após isso, será necessário utilizar os comandos:

- [git add .]

- [git commit]

- [git push]

Pull Request

O git pull é um comando do Git usado para atualizar a versão local de um repositório a partir de uma versão remota.

Como usar?

As solicitações de pull permitem que você informe outras pessoas sobre as alterações por push feitas em uma ramificação de um repositório no GitHub. Após a abertura de uma solicitação de pull, você poderá discutir e revisar as possíveis alterações com colaboradores e adicionar confirmações de acompanhamento antes que as alterações sejam mescladas na ramificação base.

1. Nesse comando é necessário apenas clicar no botão escrito [Compare & pull request] no GitHub, após ter feito o merge.
